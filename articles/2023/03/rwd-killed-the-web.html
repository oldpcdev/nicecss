How Responsive Web Design Killed the Web

Nowadays, 98% of websites are responsive. And that for a reason: The popularization of smartphones. The need to make the layout work on all devices gave birth to RWD (Responsive Web Design). Before that, designers had to duplicate work, or the site administrator would have to hire a mobile-focused web designer to make their own subdomain for the site. Example: mobile.twitter.com.

Designers got tired of using this method and started using media queries. So they ended up using a one-column layout, which is comfortable on mobile devices. However, on screens with a width of 1024px and above, reading is horrible and uncomfortable for the user.

It makes the web design look soulless. It makes no sense to focus on a layout for small screens, as the user's view is limited. Honestly, I think this is backwards.

On desktops, there are numerous layout possibilities.

That's why NiceCSS exists, to bring back that feeling of pleasure when you see a website with beautiful CSS.

I really think designers should focus more on desktops, but nobody will listen to me because I'm just a boomer.
